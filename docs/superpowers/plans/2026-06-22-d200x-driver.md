# Ulanzi D200x Driver Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add herdeck support for the Ulanzi D200x (14 LCD keys + 3 knobs + 2 side buttons) as a deck front-end reusing the grid Orchestrator, via a herdeck-native direct-HID protocol module — fully written and unit-tested **without the hardware**, against documented community-RE constants.

**Architecture:** A new pure-logic protocol module (`ulanzi_proto.py`) owns the `0x7c7c` wire format (framing, input decode, render ZIP). `D200xDriver` implements the existing `DeckDriver` interface against an injectable fake HID. Knobs/side-buttons flow through a new `on_dial` channel into the Orchestrator, which gains an additive **roaming selection cursor** and **side-button Approve/Deny** — both leaving D200/web behavior byte-identical when no dial events arrive. A standalone **probe** (Phase 0) confirms the constants on real hardware; a final **reconcile** task (HW-gated) closes the loop post-purchase.

**Tech Stack:** Python 3.12, Pillow, hidapi (`hid`), zipfile/struct (stdlib). Tests: `.venv/bin/python -m pytest`.

## Global Constraints

Bind every task. Values copied from `docs/superpowers/specs/2026-06-22-d200x-driver-design.md`.

- **Direct-HID, herdeck-native.** No `strmdck` fork. The D200 stays on `strmdck`, **unchanged**. The protocol module is shared by the probe and the driver.
- **HW-free & mergeable now.** Everything except Phase 0 (probe) and the final reconcile is unit-testable against **documented constants**, which live in **ONE clearly-marked block** in `ulanzi_proto.py`, each value tagged `# assumed from community RE — confirm via probe`.
- **Documented constants (assumed; confirm via probe):** USB `VENDOR_ID=0x2207`, `PRODUCT_ID=0x0019` (shared with the D200); framing header `b"\x7c\x7c"`, `cmd:u16` big-endian, `len:u32`, `PACKET_SIZE=1024`; commands `OUT_SET_BUTTONS=0x0001`, `OUT_SET_BRIGHTNESS=0x000a`, `OUT_PARTIALLY_UPDATE_BUTTONS=0x000d`, `IN_BUTTON=0x0101`, `IN_BUTTON_2=0x0102`, `IN_DEVICE_INFO=0x0303`; `KEY_COUNT=14`; dial indices `17,18,19`; render = ZIP(`manifest.json` keyed `f"{col}_{row}"` on a 5-column grid + per-key 196×196 PNG under `icons/`).
- **Explicit deck kind.** `kind="d200x"` is selected **explicitly** (config/env); **no auto-detect in v1** (D200/D200x share the VID/PID; the `IN_DEVICE_INFO` discriminator is unknown until the probe runs). Auto-detect is a v2 hook only.
- **No churn to D200/web.** `DeckDriver.on_dial` is a **concrete no-op default** (not abstract). The Orchestrator cursor is **additive**: with no dial events (D200/web/fake/elgato never emit them), `render()` and `on_press()` are byte-identical to today. `TileView.selected` defaults `False`; the icon/render output is unchanged when `selected is False`.
- **v2 = hooks only.** On-deck multi-option answer, refined selection scrub, and device-info auto-detect are designed in the spec but **not implemented** here — leave clearly-named seams, no logic.
- **Status panel is out of scope for v1.** The D200x's status-window addressing (button cells vs the small-window command, and which grid) is unknowable without hardware; v1 ships the standard agent-tile overview only and `render_panel` is a documented no-op. The probe dumps the mechanism and the reconcile task implements it — do not ship guessed manifest addresses.
- **Overview layout is inherited, not redefined.** `D200xDriver.slot_count()` returns 14, and the Orchestrator's existing overview reserves the last key for the `+ New` launcher (`launcher_menu` view) exactly like the D200 — so the D200x shows **up to 13 agents + the launcher key**, not 14 agents. v1 does NOT change `_agent_slots`/overview rendering; tests cover `slots=14` to lock this in.
- **Keep-alive is out of scope for v1.** Like the status panel, the D200x's keep-alive command (whether it needs periodic refresh to stay lit, and which command) is unknown without HW. v1 implements no `keep_alive_loop` (the app duck-types via `hasattr` and skips it); the probe + reconcile add it if the device sleeps/reverts. Do not ship a guessed keep-alive command.
- Commits: conventional-commit format, **no `Co-Authored-By`**.

```python
# tools/d200x_probe.py
"""Read-only Ulanzi D200x probe — run ONCE on the real device to confirm the
documented constants in src/herdeck/driver/ulanzi_proto.py.

Usage (on the real HW): .venv/bin/python tools/d200x_probe.py
Prints: enumerated HID interfaces, the IN_DEVICE_INFO response (the D200-vs-D200x
discriminator), and a live dump of raw input packets as you press keys / turn +
press each knob / press each side button. The probe is read-mostly: its ONLY write is a
single IN_DEVICE_INFO request (it never renders or changes device state); render
correctness is verified later by the full app HW gate. Reconcile any mismatch back into
ulanzi_proto's constants block (see the final reconcile task).
"""
from __future__ import annotations


def format_input_dump(data: bytes) -> str:
    b = bytes(data)
    cmd = b[2:4].hex() if len(b) >= 4 else "??"
    bd = b[8:12] if len(b) >= 12 else b[8:]
    idx = bd[1] if len(bd) >= 2 else -1
    return f"raw={b[:12].hex()} hdr={b[:2].hex()} cmd={cmd} button_data={bd.hex()} idx={idx}"


def main() -> None:  # pragma: no cover - exercised only on real hardware
    import hid

    from herdeck.driver import ulanzi_proto as P

    print("== HID enumerate ==")
    for d in hid.enumerate():
        if (d["vendor_id"], d["product_id"]) == (P.VENDOR_ID, P.PRODUCT_ID):
            print(f"  match path={d['path']!r} usage_page={d.get('usage_page')} iface={d.get('interface_number')}")
    # Open the CONTROL interface by path — Ulanzi exposes several HID interfaces and
    # opening the wrong one (e.g. the keyboard interface) fails. Try every candidate
    # in order (same shared selection logic the driver uses), falling back to the next
    # if one won't open, so a wrong CONTROL_USAGE_PAGE guess doesn't end the probe early.
    dev = None
    for path in P.select_hid_paths(hid.enumerate()):
        h = hid.device()
        try:
            h.open_path(path)
            h.set_nonblocking(True)
            dev = h
            print(f"  opened path={path!r}")
            break
        except Exception as exc:
            print(f"  failed path={path!r}: {exc}")
    if dev is None:
        raise SystemExit(f"No openable D200x interface (VID {P.VENDOR_ID:#06x} PID {P.PRODUCT_ID:#06x})")
    print("== IN_DEVICE_INFO ==")
    dev.write(b"\x00" + P.encode_command(P.IN_DEVICE_INFO, b""))
    import time

    time.sleep(0.2)
    print("  resp:", bytes(dev.read(P.PACKET_SIZE)).hex())
    print("== input dump (Ctrl-C to stop): press keys, turn+press knobs, press side buttons ==")
    while True:
        data = dev.read(P.PACKET_SIZE)
        if data:
            print(format_input_dump(bytes(data)))
        time.sleep(0.01)


if __name__ == "__main__":  # pragma: no cover
    main()
```

```python
# src/herdeck/driver/ulanzi_proto.py
"""herdeck-native Ulanzi D200x wire protocol (direct HID, no strmdck).

Pure logic — no hardware import here. Shared by the probe and D200xDriver.
"""
from __future__ import annotations

import struct

# ---------------------------------------------------------------------------
# DOCUMENTED CONSTANTS — assumed from community reverse-engineering
# (redphx/strmdck for the D200; Tyaaa-aa/Ulanzi-Deck-Linux claims D200/D200x).
# EVERY value below is "assumed from community RE — confirm via probe"
# (tools/d200x_probe.py). The final reconcile task updates anything the probe
# corrects. Do NOT treat these as verified.
# ---------------------------------------------------------------------------
VENDOR_ID = 0x2207          # shared with the D200 — confirm via probe
PRODUCT_ID = 0x0019         # shared with the D200 — confirm via probe
HEADER = b"\x7c\x7c"        # frame header — confirm via probe
PACKET_SIZE = 1024          # confirm via probe
KEY_COUNT = 14              # 14 LCD keys (D200 had 13) — confirm via probe
DIAL_INDICES = (17, 18, 19)  # the 3 knobs report as these input indices — confirm via probe

OUT_SET_BUTTONS = 0x0001
OUT_SET_SMALL_WINDOW_DATA = 0x0006
OUT_SET_BRIGHTNESS = 0x000A
OUT_PARTIALLY_UPDATE_BUTTONS = 0x000D
IN_BUTTON = 0x0101
IN_BUTTON_2 = 0x0102        # second input command (carries dial events) — confirm via probe
IN_DEVICE_INFO = 0x0303     # D200-vs-D200x discriminator — confirm via probe
CONTROL_USAGE_PAGE = 0x0C   # openable control HID interface — assumed from the D200, confirm via probe
# ---------------------------------------------------------------------------


def encode_command(cmd: int, payload: bytes) -> bytes:
    """Frame a command: header + cmd(u16 BE) + len(u32 BE) + payload."""
    return HEADER + struct.pack(">H", cmd) + struct.pack(">I", len(payload)) + payload


def chunk_packets(framed: bytes, size: int = PACKET_SIZE) -> list[bytes]:
    """Split a framed message into <= size HID write packets."""
    return [framed[i : i + size] for i in range(0, len(framed), size)] or [b""]


def select_hid_paths(enumerated, vid: int = VENDOR_ID, pid: int = PRODUCT_ID,
                     control_usage_page: int = CONTROL_USAGE_PAGE) -> list:
    """Candidate HID paths for the device, control interface first.

    Ulanzi devices expose several HID interfaces; opening the wrong one grabs the
    keyboard interface (held by the OS) and fails — so prefer the control
    usage_page, then fall back to the rest. Pure + unit-tested; shared by the probe
    and the driver so both open the same interface.
    """
    matches = [d for d in enumerated if (d.get("vendor_id"), d.get("product_id")) == (vid, pid)]
    paths = [d["path"] for d in matches if d.get("usage_page") == control_usage_page]
    paths += [d["path"] for d in matches if d.get("usage_page") != control_usage_page]
    return paths
```

```python
# add to src/herdeck/driver/ulanzi_proto.py
from dataclasses import dataclass


@dataclass
class KeyEvent:
    index: int
    pressed: bool


@dataclass
class DialEvent:
    knob: int
    delta: int  # +1 / -1 for rotation, 0 for a press
    pressed: bool


# dial_event byte meaning — assumed from community RE — confirm via probe
_DIAL_DELTA = {2: 1, 3: -1}


def decode_input(data: bytes) -> "KeyEvent | DialEvent | None":
    b = bytes(data)
    if len(b) < 12 or b[0:2] != HEADER:
        return None
    cmd = struct.unpack(">H", b[2:4])[0]
    if cmd not in (IN_BUTTON, IN_BUTTON_2):
        return None
    button_data = b[8:12]
    index = button_data[1]
    is_dial = index in DIAL_INDICES or button_data[2] == 2
    if is_dial:
        ev = button_data[3]
        return DialEvent(knob=index, delta=_DIAL_DELTA.get(ev, 0), pressed=(ev == 1))
    return KeyEvent(index=index, pressed=(button_data[3] == 0x01))
```

```python
# add to src/herdeck/driver/ulanzi_proto.py
import io as _io
import json as _json
import os as _os
import zipfile as _zipfile


@dataclass
class RenderSpec:
    label: str | None = None
    png_path: str | None = None


_BOUNDARY_INVALID = (0x00, 0x7C)


def _has_bad_boundary(data: bytes) -> bool:
    """True if an invalid byte (0x00/0x7c) lands on a 1024-byte packet boundary.

    The device's framing breaks if such a byte starts a packet; build_render_zip pads
    a leading dummy entry to shift file offsets until none do (community protocol note).
    """
    return any(data[i] in _BOUNDARY_INVALID for i in range(1016, len(data), 1024))


def build_render_zip(buttons: "dict[int, RenderSpec]") -> bytes:
    """Build the SET_BUTTONS payload: a ZIP of manifest.json + per-key PNGs.

    A leading STORED (uncompressed) dummy.txt shifts file offsets so invalid bytes
    (0x00/0x7c) avoid the 1024-byte packet boundaries. Because the dummy is stored,
    each added byte shifts later offsets by exactly 1, so the retry deterministically
    sweeps the boundary bytes. It NEVER returns a known-bad payload: if no padding in
    a full 1024-byte period clears the boundaries, it raises rather than emit bytes the
    device framing would reject.
    """
    icons: dict[str, bytes] = {}
    manifest: dict[str, dict] = {}
    for idx, spec in buttons.items():
        key = f"{idx % 5}_{idx // 5}"
        view: dict = {}
        if spec.label is not None:
            view["Text"] = spec.label
        if spec.png_path:
            name = _os.path.basename(spec.png_path)
            with open(spec.png_path, "rb") as fh:
                icons[f"icons/{name}"] = fh.read()
            view["Icon"] = f"icons/{name}"
        manifest[key] = {"State": 0, "ViewParam": [view]}
    manifest_json = _json.dumps(manifest, sort_keys=True, separators=(",", ":"))

    for pad in range(0, 1025):
        buf = _io.BytesIO()
        with _zipfile.ZipFile(buf, "w", _zipfile.ZIP_DEFLATED) as zf:
            # STORED dummy: +1 byte of padding shifts every later offset by exactly 1.
            zf.writestr(_zipfile.ZipInfo("dummy.txt"), "i" * (4 + pad), compress_type=_zipfile.ZIP_STORED)
            for name, data in icons.items():
                zf.writestr(name, data)
            zf.writestr("manifest.json", manifest_json)
        blob = buf.getvalue()
        if not _has_bad_boundary(blob):
            return blob
    raise ValueError("build_render_zip: no padding avoided invalid bytes on packet boundaries")
```

```python
# src/herdeck/driver/d200x.py
from __future__ import annotations

from collections.abc import Callable

from . import ulanzi_proto as P
from .base import DeckDriver, DialAction, PanelView, TileView

# PROVISIONAL input mapping — assumed from community RE — confirm via probe.
_KNOB_SELECT = 17
_KNOB_BRIGHTNESS = 18
_KNOB_ATTENTION = 19
_SIDE_APPROVE = 15
_SIDE_DENY = 16
_BRIGHTNESS_STEP = 10


def open_device(hid_module=None, retries: int = 5, delay: float = 1.0):  # pragma: no cover - real HW
    """Open the D200x control interface by path (enumerate + open_path + retry).

    Path selection is the shared pure helper ulanzi_proto.select_hid_paths.
    """
    import contextlib
    import time

    if hid_module is None:
        import hid as hid_module
    last = None
    for _ in range(retries):
        for path in P.select_hid_paths(hid_module.enumerate()):
            h = hid_module.device()
            try:
                h.open_path(path)
                h.set_nonblocking(True)
                return h
            except Exception as exc:  # noqa: BLE001 - try the next interface
                last = exc
                with contextlib.suppress(Exception):
                    h.close()  # never leak a handle / keep an interface busy for the retry
        time.sleep(delay)
    raise RuntimeError(f"No openable Ulanzi D200x control interface (last: {last})")


class D200xDriver(DeckDriver):
    """Ulanzi D200x driver: 14 LCD keys + 3 knobs + 2 side buttons over direct HID."""

    DEBOUNCE = 0.25  # ignore a repeat of the SAME press within this window (HW double-fire)

    def __init__(self, device, icon_provider=None, icons_dir=None, brightness: int = 80, clock=None):
        import os
        import time

        self._clock = clock or time.monotonic
        self._last_press: tuple = (None, 0.0)
        self._dev = device
        if icon_provider is None:
            # Mirror the D200: a real IconProvider so render() ships actual PNGs
            # (without one, every tile would render blank).
            from ..icons import DEFAULT_AGENT_SLUGS, IconProvider

            cache = os.path.join(os.path.expanduser("~/.cache/herdeck"), "d200x-icons")
            os.makedirs(cache, exist_ok=True)
            overrides = os.path.abspath(os.path.expanduser(icons_dir)) if icons_dir else None
            icon_provider = IconProvider(
                cache_dir=cache, slug_map=DEFAULT_AGENT_SLUGS, overrides_dir=overrides
            )
        self._icons = icon_provider
        # Absolute cache dir so build_render_zip reads PNGs by absolute path
        # (no CWD dependency, unlike the strmdck D200 path).
        self._cache_dir = getattr(icon_provider, "_cache_dir", "")
        self._brightness = brightness
        self._press_cb: Callable[[int], None] | None = None
        self._dial_cb: Callable[[DialAction], None] | None = None
        self._dev.set_nonblocking(True)
        self._send(P.OUT_SET_BRIGHTNESS, str(brightness).encode())

    def _send(self, cmd: int, payload: bytes) -> None:
        for packet in P.chunk_packets(P.encode_command(cmd, payload)):
            self._dev.write(b"\x00" + packet)

    def slot_count(self) -> int:
        return P.KEY_COUNT

    def _render_specs(self, tiles: list[TileView]) -> dict[int, P.RenderSpec]:
        import os

        specs: dict[int, P.RenderSpec] = {}
        for t in tiles:
            if t.index >= P.KEY_COUNT:
                continue
            # render_tile returns a bare filename in the cache dir; resolve to an
            # absolute path (os.path.join leaves an already-absolute path intact).
            name = self._icons.render_tile(t) if self._icons is not None else None
            png = os.path.join(self._cache_dir, name) if name else None
            specs[t.index] = P.RenderSpec(label="", png_path=png)
        return specs

    def render(self, tiles: list[TileView]) -> None:
        self._send(P.OUT_SET_BUTTONS, P.build_render_zip(self._render_specs(tiles)))

    def render_working(self, tiles: list[TileView]) -> None:
        self._send(P.OUT_PARTIALLY_UPDATE_BUTTONS, P.build_render_zip(self._render_specs(tiles)))

    def render_panel(self, panel: PanelView) -> None:
        # The status panel is OUT OF SCOPE for v1. The D200 addresses its 2-cell
        # status window as buttons 13/14 ("3_2"/"4_2") in a 5x3 grid; the D200x has
        # 14 LCD keys (0-13), so the analogous cells (14/15 -> "4_2"/"0_3") fall off
        # a 5x3 grid, and whether the D200x exposes the small window as button cells
        # at all (vs the OUT_SET_SMALL_WINDOW_DATA command) is unknown without HW.
        # Rather than ship guessed, possibly-invalid manifest addresses, v1 renders
        # the standard agent-tile overview only (the Orchestrator reserves the last
        # key for the launcher, like the D200); the probe (Task 0) dumps the panel/
        # small-window mechanism and the reconcile task (Task 12) implements render_panel.
        return None

    def on_press(self, callback: Callable[[int], None]) -> None:
        self._press_cb = callback

    def on_dial(self, callback: Callable[[DialAction], None]) -> None:
        self._dial_cb = callback

    def _emit_dial(self, action: DialAction) -> None:
        if self._dial_cb is not None:
            self._dial_cb(action)

    def _debounced(self, tag) -> bool:
        """True if `tag` (a key/knob press) repeats within DEBOUNCE — suppress it.

        Only presses are debounced; rotation deltas are intentional and pass through.
        """
        now = self._clock()
        last_tag, last_t = self._last_press
        if tag == last_tag and now - last_t < self.DEBOUNCE:
            return True
        self._last_press = (tag, now)
        return False

    def dispatch_input(self, event) -> None:
        if isinstance(event, P.KeyEvent):
            if not event.pressed:
                return
            if self._debounced(("key", event.index)):  # HW double-fire guard
                return
            if event.index == _SIDE_APPROVE:
                self._emit_dial(DialAction.APPROVE)
            elif event.index == _SIDE_DENY:
                self._emit_dial(DialAction.DENY)
            elif event.index < P.KEY_COUNT and self._press_cb is not None:
                self._press_cb(event.index)
            return
        if isinstance(event, P.DialEvent):
            # Handle ONLY the three configured knobs; an unknown/misdecoded index must
            # never move or focus the cursor.
            if event.knob == _KNOB_BRIGHTNESS:
                if event.delta:
                    self._brightness = max(0, min(100, self._brightness + event.delta * _BRIGHTNESS_STEP))
                    self._send(P.OUT_SET_BRIGHTNESS, str(self._brightness).encode())
                return
            if event.knob == _KNOB_SELECT:
                if event.pressed:
                    if not self._debounced(("dial", event.knob)):  # HW double-fire guard
                        self._emit_dial(DialAction.SELECT_PRESS)
                elif event.delta > 0:
                    self._emit_dial(DialAction.SELECT_NEXT)
                elif event.delta < 0:
                    self._emit_dial(DialAction.SELECT_PREV)
                return
            if event.knob == _KNOB_ATTENTION:
                if event.pressed:
                    if not self._debounced(("dial", event.knob)):
                        self._emit_dial(DialAction.SELECT_PRESS)  # press the attention knob to focus
                elif event.delta:
                    self._emit_dial(DialAction.ATTENTION_NEXT)
                return
            return  # unknown knob index -> ignore

    async def run_reader(self) -> None:  # pragma: no cover - exercised on real HW
        import asyncio

        while True:
            data = self._dev.read(P.PACKET_SIZE)
            event = P.decode_input(bytes(data)) if data else None
            if event is not None:
                self.dispatch_input(event)
            await asyncio.sleep(0.005)

    def close(self) -> None:
        try:
            self._dev.close()
        except Exception:
            pass
```

```python
# add the handler:
    def on_dial(self, action: DialAction) -> list[Command]:
        if self._launcher or self._profile_menu:
            return []  # a modal menu owns the deck — dials (incl. Approve/Deny) do nothing,
            #            so a side-button can never fire an invisible action on a hidden view
        if action is DialAction.SELECT_NEXT:
            return self.cursor_move(1)
        if action is DialAction.SELECT_PREV:
            return self.cursor_move(-1)
        if action is DialAction.ATTENTION_NEXT:
            return self.cursor_attention_next()
        if action is DialAction.SELECT_PRESS:
            return self.cursor_press()
        if action in (DialAction.APPROVE, DialAction.DENY):
            return self._cursor_act(approve=action is DialAction.APPROVE)
        return []

    def _cursor_act(self, approve: bool) -> list[Command]:
        key = self._cursor
        if key is None:
            return []
        if self._drill is not None:
            return []  # during drill, approval goes through the drill's own tiles —
            #            never act on a possibly-stale cursor prompt from overview
        agent = self._agents.get(key)
        # Binary-only gating (mirrors the elgato front-end): blocked + online +
        # prompt read + NO numbered options. Never approve blind.
        if agent is None or agent.status is not Status.BLOCKED:
            return []
        if key.server_id in self._down:
            return []
        if not self._cursor_detection.strip() or layout.parse_options(self._cursor_detection):
            return []
        profile = self._profile_for(key)
        keys = profile.approve if approve else profile.deny
        if not keys:
            return []
        # Preserve the existing arm-then-confirm safety path (same as _press_drill): if
        # this action requires confirmation, the FIRST side-button press only arms; a
        # second identical press confirms. A single press can never bypass require_confirm_for.
        action_id = "approve" if approve else "deny"
        if action_id in self.config.safety.require_confirm_for and self._pending_confirm != (action_id, key):
            self._pending_confirm = (action_id, key)
            return []
        self._pending_confirm = None
        # Consume the prompt so a repeat side-button press (after debounce, before the
        # async state update) cannot send the same approve/deny keys twice. cursor_act
        # refuses until a fresh read lands; the agent normally leaves BLOCKED anyway.
        self._cursor_detection = ""
        return [Command("act_if_blocked", key.server_id, key.pane_id, keys=keys)]
```
