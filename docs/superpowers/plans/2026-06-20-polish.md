# Feature polish — design + plan

**Goal:** Two adoption-focused polish wins: (1) the drill-in panel surfaces the
agent's prompt *question* clearly instead of wasting lines on option text that is
already shown on the tiles, and (2) more agent types are launchable from "+ New".

## Design decisions (made; implement these)

### 1. Richer drill prompt display (`layout.panel_detail`)
Today `panel_detail(agent, text)` shows the first 2 non-empty lines verbatim — so
for a numbered prompt the panel wastes space repeating "1. Yes / 2. No", which the
drill **tiles** already render as buttons. Change it to show the **question/prose**:
- Use `parse_options(text)` to know which lines are numbered options.
- Build `lines` from the non-empty lines that are NOT option lines (the question /
  context), up to **3** lines, each truncated to ~80 chars.
- If the prompt is ALL options (no prose), fall back to the first non-empty line so
  the panel is never empty.
- Title unchanged: `f"{agent.agent_type}: {agent.label}"`. Color unchanged.

This keeps the question on the panel and the choices on the tiles — no duplication.

### 2. More startable agents (`config.DEFAULT_START_PROFILES`)
Add `cursor`, `gemini`, `opencode` (icons already exist in
`icons.DEFAULT_AGENT_SLUGS`) so they appear on the "+ New" launcher:
```python
DEFAULT_START_PROFILES = {
    "claude": ["claude"], "codex": ["codex"],
    "cursor": ["cursor-agent"], "gemini": ["gemini"], "opencode": ["opencode"],
}
```
(Use the conventional CLI command for each; a comment noting they can be overridden
in config `[start_profiles]`.) Do NOT invent `answer_profiles` keys for these — the
generic numbered-option path + the `default` profile already cover them, and
guessing approve/deny keystrokes per CLI risks shipping wrong ones.

   Test (test_layout.py):
   ```python
   from herdeck.layout import panel_detail
   from herdeck.model import AgentKey, AgentState, Status
   def test_panel_detail_shows_question_not_option_lines():
       a = AgentState(AgentKey("s","p"), "claude", "api", Status.BLOCKED)
       pv = panel_detail(a, "Do you want to proceed?\n1. Yes\n2. No")
       assert pv.lines == ["Do you want to proceed?"]
   def test_panel_detail_all_options_falls_back_to_first_line():
       a = AgentState(AgentKey("s","p"), "claude", "api", Status.BLOCKED)
       pv = panel_detail(a, "1. Yes\n2. No")
       assert pv.lines and "1." in pv.lines[0]
   ```
